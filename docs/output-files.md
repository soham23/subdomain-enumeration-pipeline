# Generated Output Files

The pipeline stores all generated files under a domain-specific output directory.

Example:

```text
outputs/example.com/
```

The following files are created during execution:

| File             | Generated By | Description                                                                                          |
| ---------------- | ------------ | ---------------------------------------------------------------------------------------------------- |
| `sublistr.txt`   | Sublist3r    | Raw passive enumeration results from Sublist3r.                                                      |
| `subfinder.txt`  | Subfinder    | Raw passive enumeration results from Subfinder.                                                      |
| `passive.txt`    | Pipeline     | Deduplicated combination of `sublistr.txt` and `subfinder.txt`.                                      |
| `puredns.txt`    | PureDNS      | Valid subdomains discovered through DNS bruteforcing.                                                |
| `pre_altdns.txt` | Pipeline     | Seed subdomains selected for alternate hostname generation.                                          |
| `gen_subs.txt`   | AltDNS       | Generated hostname permutations before validation. *(Deleted during cleanup.)*                       |
| `massdns.out`    | MassDNS      | Raw DNS validation output from MassDNS. *(Deleted during cleanup.)*                                  |
| `altdns.txt`     | Pipeline     | Valid alternate hostnames extracted from the MassDNS output.                                         |
| `final.txt`      | Pipeline     | Final deduplicated list containing all discovered subdomains. Created only if requested by the user. |

## Cleanup

The pipeline automatically removes the following files after execution:

* `gen_subs.txt`
* `massdns.out`

These files can become very large during alternate hostname generation and DNS validation. Since the useful results are preserved in `altdns.txt`, the raw intermediate files are removed to reduce disk usage and keep the output directory organized.

## Notes

* Each target domain receives its own output directory.
* Intermediate files are intentionally preserved (except temporary files) to make it easy to inspect the output of each stage or troubleshoot the pipeline.
* `final.txt` is generated only after user confirmation.
